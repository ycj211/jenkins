Imposta un trigger in modo che quando la compilazione di altri progetti termina venga pianificata una nuova compilazione per questo progetto. Ciò è conveniente, ad esempio, per eseguire test estesi dopo il termine di una compilazione.

Quest'opzione completa la sezione "Compila altri progetti" nelle "Azioni di post compilazione" di un progetto upstream, ma è preferibile qualora si voglia configurare il progetto downstream.
