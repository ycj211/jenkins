Utilizza il container servlet per autenticare gli utenti, così come definito dalle specifiche della servlet. Ciò storicamente è il comportamento tenuto da Jenkins fino alla versione 1.163. È utile principalmente nelle seguenti situazioni:
  1. Si è usato Jenkins in una versione precedente alla 1.164 e si vorrebbe continuare a mantenere tale comportamento.
  2. Si è già configurato il proprio container con l'area di autenticazione corretta e si preferisce che Jenkins la utilizzi, semplicemente. (A volte il container offre una documentazione migliore o implementazioni personalizzate per connettersi a un realm utente specifico).
