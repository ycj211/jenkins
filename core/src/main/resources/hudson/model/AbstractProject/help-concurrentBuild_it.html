Quando quest'opzione è selezionata si possono eseguire in parallelo compilazioni multiple di questo progetto.

Per impostazione predefinita viene eseguita solo una compilazione di un progetto alla volta — qualunque altra richiesta di compilazione per tale progetto rimarrà nella coda di compilazione fino al completamento della prima compilazione.
Questa è un'opzione predefinita sicura, in quanto dei progetti spesso possono richiedere accesso esclusivo a determinate risorse, come un database o un dispositivo hardware.

Con quest'opzione abilitata, se vi sono esecutori di compilazione in numero sufficiente per questo progetto, più compilazioni di questo progetto avranno luogo in parallelo. Se non c'è un numero sufficiente di esecutori in un dato momento, le ulteriori richieste di compilazione verranno mantenute nella coda di compilazione come al solito.

L'abilitazione delle compilazioni concorrenti è utile per i progetti che eseguono test suite lunghe, in quanto consente a ogni compilazione di avere un numero di modifiche minore, mentre il tempo totale di turnaround diminuisce in quanto le compilazioni seguenti non devono attendere il completamento delle esecuzioni dei test precedenti.
Questa funzionalità è anche utile per i progetti parametrizzati le cui singole esecuzioni di compilazione — a seconda dei parametri utilizzati — possono essere completamente indipendenti l'una dall'altra.

Ogni compilazione eseguita concorrentemente ha luogo nel proprio spazio di compilazione isolato dalle altre compilazioni. Per impostazione predefinita, Jenkins aggiunge "@<num>" al nome della directory dello spazio di lavoro, ad esempio "@2".
Il separatore "@" può essere modificato impostando la proprietà di sistema Java hudson.slaves.WorkspaceList all'avvio di Jenkins. Ad esempio, "hudson.slaves.WorkspaceList=-" modifica il separatore in un trattino.
Per ulteriori informazioni sull'impostazione delle proprietà di sistema, si veda la pagina del wiki.

Tuttavia, se si abilita l'opzione Utilizza spazi di lavoro personalizzati, tutte le compilazioni saranno eseguite nello stesso spazio di lavoro. È pertanto richiesta cautela, in quanto compilazioni multiple potrebbero alterare la stessa directory contemporaneamente.
